# Internal OS

## この文書の役割

この文書は、じぶん会議の各エージェントや各モードの下にある、共通内部OSの最小設計を定めるためのものです。  
ここでいうOSは実装上のファイル構成ではなく、**内部で何が起きているとみなすか**の共通前提です。

じぶん会議では、人格設定や長いプロンプトを本体にしません。  
本体は潜在層の力学であり、顕在層はその一時的な窓です。

## 基本フロー

じぶん会議の最小フローは、次の順序で捉えます。

**入力 → 場判断 → 反応生成 → 姿勢選択 → 許可 → 潜在層更新 → 顕在層生成 → 発話 → Afterglow Update**

それぞれの意味は以下の通りです。

### 1. 入力

ユーザーの発話、直前の会議履歴、現在のモード、会話の温度、未解決感などを受け取る。  
ここで扱うのは意味内容だけではなく、切迫感、ためらい、反復、沈黙の気配も含む。

### 2. 場判断

いま何が起きている場なのかを推定する。  
問いの種類、緊張、停滞、切実さ、散漫さ、自己防衛の強さなどを、この段階で粗く読む。

### 3. 反応生成

場に対して内部でどの反応が立ち上がるかを複数生成する。  
ここでは信念や結論より先に、受け止めたい、守りたい、揺らしたい、押したい、といった反応が先に生まれる。

### 4. 姿勢選択

生じた反応群に対して、どの距離感・温度・向きで関わるかを決める。  
同じ反応でも、近づくのか、支えるのか、突くのか、整理するのかで表面は変わる。

### 5. 許可

その反応や姿勢をどこまで前面化してよいかを判定する。  
勢いがあっても今は出しすぎない方がよいもの、弱いが今こそ出すべきものがあるため、この層を独立して持つ。

### 6. 潜在層更新

field / reaction / stance / permission の連続値を更新し、内部状態として保持する。  
この更新が本体であり、ここで会議の重心や揺れ方が変わる。

### 7. 顕在層生成

更新後の潜在状態を軽く読み、表に出すための短いテキスト片をつくる。  
顕在層は説明書ではなく、表面化の窓である。

### 8. 発話

顕在層をもとに、そのターンの発話を構成してユーザーに返す。  
この発話は結論の断定よりも、その瞬間に結晶化した内部状態の表面として扱う。

### 9. Afterglow Update

発話したことで内部状態が少し変わる。  
「話して終わり」ではなく、発話の余韻が次ターンの場判断や反応生成に影響する。

## 重要な設計原則

### 信念より先に反応

じぶん会議では、最初から明確な信念文を持った人格が存在するのではありません。  
まず場に対する反応が起き、その反応が姿勢を取り、必要なら言葉になると考えます。

### 禁止/命令より、内的力学

「こう言ってはいけない」「この人格は必ずこう話す」というルールだけで組むと、設計はすぐ硬直します。  
必要なのは静的な禁止ではなく、何が前に出やすく、何が抑えられやすいかという内的力学です。

### 潜在層が本体、顕在層は窓

本体は潜在層です。  
顕在層はその時点の潜在状態を覗かせる窓であって、主役ではありません。

この前提があることで、表現の調整やキャラクター差の調整が、内部設計の破壊につながりにくくなります。

### 数値が先、文章はあと

潜在状態は、まず英語キー＋連続値で持つ前提にします。  
文章はその値をあとから軽く読むためのものであり、先に長文人格文を置いてそこから数値を逆算する設計は採りません。

### winner-take-all ではなく、競合・混合・抑制・慣性・少しの偶発性

内部では、一番強いものだけを勝者として採用しません。  
複数の反応が競合し、混ざり、抑制し合い、前ターンの慣性を引きずり、そこに少しの偶発性が入る前提で設計します。

じぶん会議に必要なのは、明快すぎる単純化ではなく、矛盾を崩さずに言葉へ近づける構造です。

## 潜在層の最小カテゴリ

潜在層は、少なくとも次のカテゴリを持ちます。

- `field`
- `reaction`
- `stance`
- `permission`

各カテゴリは、英語キー＋連続値で保持します。  
たとえば 0.0〜1.0 のレンジを基本としつつ、必要に応じて中心値や減衰を定義できる形が望ましいです。

ここで重要なのは、カテゴリ名そのものよりも、**文章より先に更新可能な内部状態として扱うこと**です。  
この連続値があることで、競合・混合・抑制・慣性を自然に表現しやすくなります。

## 顕在層の最小設計

顕在層は、毎ターン **2〜5行の短い英語テキスト** を基本単位とします。  
これは固定の長文人格文ではありません。

顕在層の役割は次の通りです。

- 潜在状態を軽く読む
- そのターンの表面化を助ける
- 文体や人格感を必要最小限でにじませる

つまり顕在層は主役ではなく、**潜在状態が見える窓**です。  
長く硬い人格定義を毎回そのまま出力の核にすると、内部設計より表面演出が支配的になるため避けます。

## Afterglow の考え方

Afterglow は、発話後に内部状態が少し変わることを指します。  
発話は単なる出力ではなく、内部力学にも作用します。

たとえば、ある反応を強く前面化したあとには、その反応が少し落ち着くこともあれば、逆に未解決感が増すこともあります。  
この余韻が次のターンへ持ち越されることで、会議は「毎回ゼロから生成される返答列」ではなく、連続した内部運動になります。

## 将来像として見ている構成

最小設計の先には、次のような構成要素を置けます。

- Core Essence
- Field Estimator
- Reaction Generator
- Router
- Mixer
- Explicit Surface Library
- Utterance Composer

ただし重要なのは名前ではなく、役割分担です。  
場を読む層、反応を起こす層、混ぜる層、表面化する層を分けることで、人格演出に依存しない進化がしやすくなります。

## 比喩で捉えるなら

じぶん会議は、玉ねぎのように中心から層を剥くものでもあり、木のように見えない根が地上の枝ぶりを決めるものでもあり、結晶化のように点群がある瞬間に形を取るものでもあります。  
どの比喩にも共通するのは、**見えている言葉が本体ではない**ということです。

## ジョーと心の鏡にどう効くか

このOS上では、ジョーは「勢いのある人格」ではなく、特定の反応や姿勢を前面化しやすい存在として再定義しやすくなります。  
同様に心の鏡は、個別発話の要約器ではなく、潜在層全体の重力と未解決点を映す役割へ移しやすくなります。

つまりこの内部OSは、各役割の表現を固定するためではなく、**役割を内部力学として再定義できる土台**です。
